# AsyncCore-TCP-Server

> **Low-Level 최적화와 모듈화 구조를 적용한 고성능 비동기 게임 서버 엔진**

C# `async/await` 비동기 소켓 통신을 기반으로, 가비지 컬렉터(GC) 부하를 최소화하는 **Zero-GC 최적화**와 `Dictionary` 기반의 **$O(1)$ 패킷 핸들러 매핑**, TCP 파편화를 방어하는 **커스텀 링 버퍼(Ring Buffer)**를 구현한 멀티스레드 게임 서버 인프라입니다.

---

## 🏗️ 솔루션 아키텍처
* **Server**: 비동기 패킷 처리, 메모리 풀링 및 컨텐츠 로직을 담당하는 핵심 엔진
* **ClientMock**: `Task.WhenAll` 기반의 고동시성 스트레스 테스트 봇 (가상 유저 100명 시나리오)
* **Unity Client**: 고정 크기 구조체 마샬링을 통한 서버와의 실시간 좌표 동기화
* **Common**: 서버-클라이언트 간 명세를 공유하는 패킷 구조체 라이브러리

---

## 🚀 핵심 기술 및 최적화 포인트

| 최적화 분야 | 기술 스택 / 기법 | 달성 효과 |
| :--- | :--- | :--- |
| **통신 인프라** | TCP Socket + `async/await` | 고동시성 비동기 처리 |
| **직렬화** | `StructLayout` + `Marshal` | C++ 수준의 로우레벨 고속 직렬화 |
| **메모리 최적화** | `.NET ArrayPool` | 힙 할당 최소화 (Zero-GC 달성) |
| **패킷 라우팅** | `Dictionary<ushort, Action>` | $O(1)$ 시간 복잡도의 고속 핸들링 |
| **스트림 제어** | 커스텀 링 버퍼 (Ring Buffer) | TCP 뭉침/쪼개짐(Fragmentation) 방어 |

---

## 📈 프로젝트 성장 스토리라인

- **[1단계] 엔진 기초:** 비동기 소켓 통신 및 패킷 헤더 구조체 설계
- **[2단계] 제로 가비지:** `ArrayPool`과 `Buffer.BlockCopy`를 통한 GC 부하 원천 차단
- **[3단계] 인프라 고도화:** TCP 파편화/뭉침 현상을 방어하는 커스텀 링 버퍼 구현
- **[4단계] 아키텍처 최적화:** $O(1)$ 패킷 핸들러 매핑 시스템 구축
- **[5단계] 시뮬레이션:** 가상 유저 100인 동시성 시나리오 스트레스 테스트 통과
- **[6단계] 클라이언트 연동:** 유니티 엔진과 서버 간 실시간 위치 동기화(MoveNotify) 완료

---

## 💡 면접 대비 핵심 기술 노트

1. **데이터 전송:** `StructLayout(Pack=1)`을 통해 서버와 클라이언트 간 메모리 구조를 일치시켰으며, `Marshal`을 활용해 오버헤드 없이 바이트 배열로 직렬화하여 포인터 연산급의 성능을 확보했습니다.
2. **비동기 처리:** `async/await`를 통해 스레드 차단 없이 다수의 클라이언트 요청을 효율적으로 처리하며, 4바이트 헤더를 먼저 확인하여 패킷을 분리하는 '영수증 스캔 방식'으로 TCP 문제를 해결했습니다.
3. **고속 라우팅:** `Dictionary`를 사용하여 조건문(`if-else`) 없이 패킷 ID를 즉시 담당 핸들러로 전달하는 하이패스 구조를 구현했습니다.
4. **동시성 관리:** `ConcurrentDictionary`를 통해 멀티스레드 환경에서 유저 세션 정보와 위치 데이터를 안전하게 동기화하고 관리합니다.

## 🎮 개발 성과 (유니티 연동)
* **실시간 동기화:** 유니티 클라이언트에서 보낸 이동 패킷을 서버가 수신하여 처리하고, 그 결과를 서버 로그 및 유니티 화면에 실시간으로 반영하는 데 성공했습니다.
* **시뮬레이션:** 100인의 가상 봇이 월드에 접속하여 이동/채팅하는 스트레스 테스트를 통해 엔진의 안정성을 검증했습니다.
