# AsyncCore-TCP-Server

> **Low-Level 최적화와 모듈화 구조를 적용한 고성능 비동기 게임 서버 엔진**

C# `async/await` 비동기 소켓 통신을 기반으로, 가비지 컬렉터(GC) 부하를 최소화하는 **Zero-GC 최적화**와 `Dictionary` 기반의 **O(1) 패킷 핸들러 매핑**, TCP 파편화를 방어하는 **커스텀 링 버퍼(Ring Buffer)**를 구현한 멀티스레드 게임 서버 인프라입니다.

---

## 🏗️ 솔루션 아키텍처

책임을 명확히 분리하기 위해 3개 모듈로 구조화했습니다.

| 모듈 | 역할 |
|---|---|
| **Server** | 비동기 패킷 수신·역직렬화, 메모리 풀링, 컨텐츠 로직을 담당하는 핵심 엔진 |
| **ClientMock** | `Task.WhenAll` 기반 고동시성 스트레스 테스트 봇 (100인 시나리오 테스트) |
| **Unity Client** | 서버와 실시간 좌표 동기화 및 물리 연산 처리 |
| **Common** | 서버-클라이언트 공유 패킷 명세 및 링 버퍼 로직 |

## 🚀 핵심 기술 및 최적화 포인트

| 최적화 분야 | 기술 스택 / 기법 | 달성 효과 |
|---|---|---|
| 통신 인프라 | TCP Socket + `async/await` | 스레드 효율 극대화 (고동시성 처리) |
| 직렬화 | `StructLayout(Pack=1)` + `Marshal` | 고속 직렬화 및 클라이언트-서버 규격 일치 |
| 메모리 관리 | `.NET ArrayPool<byte>` | 힙 할당 최소화 (Zero-GC) |
| 패킷 라우팅 | `Dictionary<ushort, Action<...>>` | O(1) 시간 복잡도 핸들링 |
| 데이터 정합성 | 커스텀 링 버퍼 | TCP 파편화/뭉침 방어 및 데이터 복원 |
| 멀티스레드 | `ConcurrentDictionary` | 세션 안전 관리 |

## 🛠️ [UPDATE] 주요 업데이트 (v1.1)

1. **상하좌우 위치 동기화 완벽 구현**
   - 횡스크롤 기준의 기존 코드를 탑다운(Plane) 환경에 최적화된 로직으로 전면 수정.
   - `MovePacket` 수신 시 `PosY` 값을 정상적으로 파싱하고 서버에 저장하도록 동기화 로직 보완.
2. **서버-클라이언트 패킷 브로드캐스트 시스템 구축**
   - 서버가 수신한 이동 패킷을 나를 제외한 모든 접속 유저에게 실시간으로 브로드캐스트하여 멀티플레이어 환경 구현.
3. **네트워크 안정성 고도화**
   - 비동기 수신 루프(`HandleClientAsync`)에서 패킷 파싱 중 발생하던 버퍼 오염 문제 해결.
   - `ArrayPool` 버퍼에서 패킷 헤더/바디 크기만큼만 정확히 슬라이싱하는 정밀 파싱 로직 적용.
4. **봇 시나리오 고도화**
   - 로그인 응답(`LoginResponse`) 대기 로직 추가 및 서버로부터 발급받은 정확한 `PlayerId` 동기화 처리.

## 🔍 TCP 파편화 문제 해결
TCP는 스트림 프로토콜로, 패킷이 쪼개지거나 뭉쳐서 도착할 수 있습니다. 이를 위해 **링 버퍼**를 활용하여 데이터가 모두 모일 때까지 기다린 후, 규격에 맞는 정확한 크기만큼만 잘라내어 핸들러로 전달하는 안전한 수신 구조를 확립했습니다.
